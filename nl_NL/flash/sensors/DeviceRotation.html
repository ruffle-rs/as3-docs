Klassepublic class DeviceRotation
OverervingDeviceRotation EventDispatcher Object

Taalversie: ActionScript 3.0
Runtimeversies: AIR 26.0

De DeviceRotation-klasse verzendt gebeurtenissen op basis van de activiteit die door de versnellingsmeter en de gyroscoopsensoren van het apparaat wordt gedetecteerd. Deze gegevens vertegenwoordigen de rol, instelhoek, rotatie rond de y-as en viertallen van het apparaat. Wanneer het apparaat draait, detecteren de sensoren deze rotatie en worden deze gegevens geretourneerd. De DeviceRotation-klasse biedt methoden om met een query na te gaan of de gebeurtenishandler voor DeviceRotation al dan niet wordt ondersteund, en om de snelheid in te stellen waarmee deze gebeurtenissen worden verzonden.

Opmerking: Gebruik de eigenschap DeviceRotation.isSupported om te testen of de runtimeomgeving ondersteuning biedt voor deze functie. De DeviceRotation-klasse en de bijbehorende elementen zijn toegankelijk voor meerdere platformen en apparaten, maar dat betekent niet dat de handler bij uitvoering altijd wordt ondersteund. Er zijn enkele gevallen waar deze handler niet wordt ondersteund, zoals de Android-versie. Hier moet u de correcte functionaliteit dus controleren met gebruik van de eigenschap DeviceRotation.isSupported. Als DeviceRotation.isSupported in de runtime de waarde true heeft, is er momenteel ondersteuning voor de deze functie.

AIR-profielondersteuning: deze functie wordt alleen ondersteund op mobiele apparaten. Deze functie wordt niet ondersteund op het bureaublad of bij AIR for TV-apparaten. Zie AIR-profielondersteuning voor meer informatie over API-ondersteuning voor meerdere profielen.

    public static function get isSupported():Boolean

Voorbeeld  ( Hoe dit voorbeeld te gebruiken )
In het volgende voorbeeld wordt de DeviceRotation.isSupported-eigenschap gebruikt om te testen op DeviceRotation-ondersteuning tijdens runtime. Als de huidige omgeving ondersteuning biedt voor de functie DeviceRotation, wordt er een gebeurtenislistener toegevoegd aan het DeviceRotation-object en vult de hieraan gekoppelde handler het tekstveld met de tijdstempel en de waarden voor de rol, de instelhoek en de rotatie rond de y-as. Als dit niet het geval is, geeft het tekstveld aan dat de huidige omgeving geen ondersteuning biedt voor de functie. 
var myTextField:TextField = new TextField();
myTextField.width = 200;
addChild(myTextField);
var deviceRotation = new DeviceRotation();
var isSupported:Boolean = DeviceRotation.isSupported;
checksupport();

function checksupport():void {
    if (isSupported) {
        myTextField.text = "DeviceRotation feature supported";
        deviceRotation.addEventListener(DeviceRotationEvent.UPDATE,deviceRotUpdateHandler);
    } else {
        myTextField.text = "DeviceRotation feature not supported";
    }
}

function deviceRotUpdateHandler(evt:DeviceRotationEvent):void {
    myTextField.text = String("at: " + evt.timestamp + "\n" + "Roll: " + evt.roll + "\n" + "Yaw: " + evt.yaw + "\n" + "Pitch: " + evt.pitch);

}

    setRequestedUpdateInterval

()methode
public function setRequestedUpdateInterval(interval:Number):void

Taalversie: ActionScript 3.0
Runtimeversies: AIR 26.0

De methode setRequestedUpdateInterval wordt gebruikt om het tijdsinterval voor updates in te stellen. Het tijdsinterval wordt gemeten in milliseconden. Het update-interval wordt alleen gebruikt als een hint om batterijvermogen te conserveren. De werkelijke tijd tussen vectorupdates voor apparaatrotatie kan groter of kleiner zijn dan deze waarde. Elke wijziging in de update-interval be√Ønvloedt alle geregistreerde listeners. U kunt de DeviceRotation-klasse gebruiken zonder de setRequestedUpdateInterval()-methode op te roepen. In dat geval ontvangt de toepassing updates op basis van de standaardinterval van het apparaat.

Parameters

interval:Number — Het aangevraagde update-interval. Als de waarde voor interval is ingesteld op 0, wordt het minimaal ondersteunde bijwerkingsinterval gebruikt.


Gegenereerde uitzondering
ArgumentError — Het opgegeven interval is minder dan nul.

Taalversie: ActionScript 3.0
Runtimeversies: AIR 26.0

De update-gebeurtenis wordt verzonden wanneer het apparaat wordt gedraaid zoals gedetecteerd door de gecombineerde lezingen van de versnellingsmeter en gyroscoopsensoren. De gebeurtenis wordt verzonden in de volgende omstandigheden:

  • Wanneer een nieuwe listenerfunctie wordt gekoppeld via addEventListener(), wordt deze gebeurtenis eenmaal aan alle geregistreerde listeners geleverd om de huidige waarde van de apparaatrotatievector te bieden aan de hand van de rol, instelhoek en viertallen.
  • Wanneer updates van de apparaatrotatie worden verkregen via het platform op intervallen die door het apparaat zijn bepaald.
  • Wanneer de toepassing een wijziging in de apparaatrotatie overslaat (bijvoorbeeld wanneer de uitvoering wordt hervat nadat deze inactief is geweest).
