LiveCycle Reader Extensions

  • Reader Extensions (no security and credential security)

  • Reader Extensions (password security)

  • Reader Extensions (policy security)

This information applies to QPACs that were provided for interacting with LiveCycle Reader Extensions 7.x.

The type of security that is applied to the PDF to which usage rights are applied determines which service operation replaces Reader Extensions actions during upgrades:

No security:
See Reader Extensions (no security and credential security)

Password security:
See Reader Extensions (password security)

Credential security:
See Reader Extensions (no security and credential security)

Policy security:
See Reader Extensions (policy security)

Reader Extensions (no security and credential security)

When the PDF document to which usage rights are applied has no security or credential security configured on it, Reader Extensions actions are replaced with Apply Usage Rights operations that the Reader Extensions service provides.

For information about property mappings that are common to all QPAC actions, see Common property mappings .

Reader Extensions (password security)

When the PDF document to which usage rights are applied has password security configured on it, Reader Extensions actions are replaced with two operations routed sequentially:

  • unlockPDFUsingPassword (Encryption service)

  • Apply Usage Rights (Reader Extensions service)

    For information about property mappings that are common to all QPAC actions, see Common property mappings .

Reader Extensions (policy security)

When the PDF document to which usage rights are applied has policy security configured on it, Reader Extensions actions are replaced with two operations routed sequentially:

  • Unlock policy protected PDF (Rights Management service)

  • Apply Usage Rights (Reader Extensions service)

    For information about property mappings that are common to all QPAC actions, see Common property mappings .
